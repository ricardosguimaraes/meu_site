## Gerenciar posts

Você pode criar, editar e excluir posts adicionando, modificando ou removendo arquivos `*.mdx` no diretório `src/app/blog/posts`.

## Frontmatter

O frontmatter é usado para definir os metadados do post: título, descrição, imagem e tags. Ele é utilizado na página do post e nas meta tags/open graph.

<CodeBlock
  marginBottom="16"
  codes={[
    {
      code: `---
title: "Alcançando um novo marco na minha carreira"
publishedAt: "2024-04-08"
image: "/images/gallery/img-02.jpg"
summary: "Toda carreira é uma jornada, cheia de desafios, crescimento e aqueles momentos marcantes que mudam nosso caminho."
tag: "Jornal"
---`,
      language: "tsx",
      label: "src/app/blog/posts/post-1.mdx",
    },
  ]}
/>

## Sintaxe MDX

Você pode usar MDX para escrever o conteúdo dos seus posts. Isso permite criar conteúdos ricos e dinâmicos com pouco código. Alguns elementos MDX serão automaticamente transformados em componentes Once UI para melhor integração visual e funcionalidade adicional.

## Componentes personalizados

Você pode usar componentes personalizados em arquivos MDX, mas é necessário importá-los primeiro em `src/components/mdx.tsx`.

<CodeBlock
  marginBottom="16"
  highlight="11-12"
  codes={[
    {
      code: `const components = {
    p: createParagraph as any,
    h1: createHeading(1) as any,
    h2: createHeading(2) as any,
    h3: createHeading(3) as any,
    h4: createHeading(4) as any,
    h5: createHeading(5) as any,
    h6: createHeading(6) as any,
    img: createImage as any,
    a: CustomLink as any,
    Table,
    CodeBlock,
};`,
      language: "tsx",
      label: "src/components/mdx.tsx",
    },
  ]}
/>

Como pode ver, os componentes `Table` e `CodeBlock` já estão importados e disponíveis para uso. Você pode adicionar mais componentes importando-os neste arquivo e passando-os para o objeto `components`.

## Hot reload

O hot reload de arquivos MDX ainda não é suportado, mas estamos trabalhando nisso.
