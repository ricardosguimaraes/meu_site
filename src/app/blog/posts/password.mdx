## Adicionar páginas protegidas

Para ativar a proteção por senha em páginas específicas, adicione os caminhos das páginas que deseja proteger ao objeto `protectedRoutes` no arquivo `resources/config.js`.

O componente `RouteGuard` irá automaticamente gerenciar o controle de acesso dessas páginas, exigindo autenticação por senha antes de permitir o acesso.

<CodeBlock
  marginBottom="16"
  highlight="2"
  codes={[
    {
      code: `const protectedRoutes = {
  '/work/once-ui': true
};`,
      language: "tsx",
      label: "src/app/resources/config.js",
    },
  ]}
/>

O código acima garante que a página `/work/once-ui` só estará acessível após fornecer uma senha.

## Definir uma senha

A senha pode ser definida no arquivo `.env.local` para desenvolvimento local e como uma [variável de ambiente](https://vercel.com/docs/environment-variables) para produção.
